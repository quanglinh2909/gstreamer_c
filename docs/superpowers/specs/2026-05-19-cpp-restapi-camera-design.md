# C++ REST API with PostgreSQL вЂ” Design

**Date:** 2026-05-19
**Project name (CMake):** `test_gstreamer`
**Stack:** Oat++ + vcpkg + oatpp-postgresql + oatpp-swagger
**Sample entity:** `Camera (id, name, rtsp, status)`

## 1. Goal & Scope

Bootstrap a C++20 RESTful API server with proper, idiomatic Oat++ project layout, PostgreSQL connection pool, and Swagger/OpenAPI documentation. Scaffolds a full CRUD example for the `Camera` entity to demonstrate the layering.

Out of scope: authentication, logging framework, automated migrations, full test coverage. JWT / structured logging deliberately deferred (YAGNI for this scaffold).

Updates after initial design (2026-05-19):
- `Camera.id` is a **UUID** generated by Postgres (`gen_random_uuid()`), not BIGSERIAL.
- All runtime values (server host/port, DB URL, pool sizing, swagger metadata) are loaded from `config/config.json` via a `ConfigComponent`. Path resolution: argv[1] > `$CONFIG_PATH` > `config/config.json`.

## 2. Architecture Overview

Single binary started by `App.cpp`:

1. `oatpp::base::Environment::init()`
2. Instantiate `AppComponent` вЂ” registers DI beans: `ServerConnectionProvider`, `HttpRouter`, `ObjectMapper` (JSON, beautify in dev).
3. `DatabaseComponent` builds `postgresql::ConnectionProvider` from a hard-coded connection string (config file deferred), wraps it in `ConnectionPool::createShared(provider, 10, 60s)`, exposes a `CameraDb` (DbClient) bean.
4. `SwaggerComponent` mounts `/swagger/ui` and `/api-docs/oas-3.0.0.json`.
5. Router registers `CameraController`; `HttpConnectionHandler` starts listening on `0.0.0.0:8000`.

Request flow:

```
HTTP request в†’ HttpRouter в†’ CameraController (ENDPOINT macro)
                              в†“
                          CameraService (validate, business)
                              в†“
                          CameraDb (DbClient query macros) в†’ libpq в†’ Postgres
                              в†‘
                          QueryResult в†’ fetch into CameraDto
                              в†‘
                          ObjectMapper serialize в†’ HTTP response
```

## 4. Components & Responsibilities

### `App.cpp`
Entry. Initializes Oat++ environment, instantiates `AppComponent`, builds `HttpConnectionHandler`, starts `HttpConnectionProvider->serve`, traps SIGINT for graceful shutdown.

### `AppComponent.hpp`
DI beans:

- `ServerConnectionProvider` вЂ” host `0.0.0.0`, port `8000`.
- `HttpRouter`.
- `ObjectMapper` (JSON, beautify on).
- Default Oat++ `ErrorHandler`.

### `DatabaseComponent.hpp`
- `postgresql::ConnectionProvider` from connection string `postgresql://user:pass@localhost:5432/test_gstreamer`.
- `ConnectionPool::createShared(provider, maxConnections=10, maxIdleTime=60s)`.
- `CameraDb` instance (DbClient) bound to the pool.

### `SwaggerComponent.hpp`
- `oatpp::swagger::DocumentInfo` вЂ” title `test_gstreamer API`, version `1.0`.
- `oatpp::swagger::Resources::loadResources(OATPP_SWAGGER_RES_PATH)`.

### `dto/CameraDto.hpp`

```cpp
class CameraDto : public oatpp::DTO {
  DTO_INIT(CameraDto, DTO)
  DTO_FIELD(Int64,  id);
  DTO_FIELD(String, name);
  DTO_FIELD(String, rtsp);
  DTO_FIELD(String, status);
};

class CreateCameraDto : public oatpp::DTO {
  DTO_INIT(CreateCameraDto, DTO)
  DTO_FIELD(String, name);
  DTO_FIELD(String, rtsp);
  DTO_FIELD(String, status);   // default "offline" when null (set in service)
};
```

### `dto/StatusDto.hpp`

```cpp
class StatusDto : public oatpp::DTO {
  DTO_INIT(StatusDto, DTO)
  DTO_FIELD(Int32,  statusCode);
  DTO_FIELD(String, message);
};
```

### `db/CameraDb.hpp` вЂ” 5 queries

```
createCamera(name, rtsp, status)
  INSERT INTO cameras(name, rtsp, status) VALUES (:name, :rtsp, :status) RETURNING *;

getCameraById(id)
  SELECT * FROM cameras WHERE id = :id LIMIT 1;

getAllCameras(limit, offset)
  SELECT * FROM cameras ORDER BY id LIMIT :limit OFFSET :offset;

updateCamera(id, name, rtsp, status)
  UPDATE cameras SET name=:name, rtsp=:rtsp, status=:status WHERE id=:id RETURNING *;

deleteCamera(id)
  DELETE FROM cameras WHERE id=:id;
```

### `service/CameraService.hpp`

5 methods mirroring the queries. Validation rules:

- `name` not empty.
- `rtsp` not empty, must start with `rtsp://`.
- `status` must be one of `online`, `offline`, `error`. Default to `offline` if null on create.

Error mapping uses `OATPP_ASSERT_HTTP(cond, code, msg)`.

### `controller/CameraController.hpp`

Base path `/cameras`. One handler per endpoint, each annotated with `ENDPOINT_INFO` for Swagger.

## 5. REST Endpoints

| Method | Path              | Body              | Success            | Failures      |
|--------|-------------------|-------------------|--------------------|---------------|
| GET    | `/cameras`        | вЂ”                 | `200 List<Camera>` | 500           |
| GET    | `/cameras/{id}`   | вЂ”                 | `200 Camera`       | 404, 500      |
| POST   | `/cameras`        | `CreateCameraDto` | `201 Camera`       | 400, 500      |
| PUT    | `/cameras/{id}`   | `CreateCameraDto` | `200 Camera`       | 400, 404, 500 |
| DELETE | `/cameras/{id}`   | вЂ”                 | `200 StatusDto`    | 404, 500      |

### Data flow вЂ” POST /cameras

1. Controller: ObjectMapper deserializes body в†’ `CreateCameraDto`. Malformed JSON в†’ Oat++ returns 400 automatically.
2. Service: validates fields; defaults `status` to `"offline"` if null.
3. Service calls `m_db->createCamera(...)`; checks `result->isSuccess()`.
4. Fetches the inserted row from `QueryResult` into `CameraDto`.
5. Controller returns `createDtoResponse(Status::CODE_201, dto)`.

### Data flow вЂ” GET /cameras/{id}

1. Controller binds path param `id` as `Int64`.
2. Service calls `m_db->getCameraById(id)`.
3. If `result->isSuccess() == false` в†’ 500. If `result->hasMoreToFetch() == false` в†’ 404.
4. Fetch and return `CameraDto`.

### Connection pool behavior

- Each query borrows one connection from the pool; returned automatically when `QueryResult` is destroyed.
- Max 10 connections, idle timeout 60s.
- Pool is thread-safe; Oat++ handler threads request connections independently.

## 6. Error Handling

Centralized via Oat++ default `ErrorHandler`, which serializes `HttpError` thrown by `OATPP_ASSERT_HTTP` into:

```json
{ "statusCode": 404, "message": "Camera not found" }
```

| Situation | Detected in | HTTP | Message |
|---|---|---|---|
| JSON body malformed | ObjectMapper | 400 | "Can't parse body" |
| DTO missing required field | Service | 400 | "Field X required" |
| `rtsp` not starting with `rtsp://` | Service | 400 | "Invalid rtsp url" |
| `status` not in {online, offline, error} | Service | 400 | "Invalid status value" |
| `id` not found | Service (`hasMoreToFetch() == false`) | 404 | "Camera not found" |
| Postgres query failure | Service (`isSuccess() == false`) | 500 | "Database error: ..." |
| Pool exhausted / timeout | oatpp internal | 500 | (default) |

No broad `try/catch`; the Oat++ pattern is to throw via `OATPP_ASSERT_HTTP` and let the `ErrorHandler` convert.

## 7. SQL Schema

`sql/001_init_cameras.sql`:

```sql
CREATE EXTENSION IF NOT EXISTS pgcrypto;

CREATE TABLE IF NOT EXISTS cameras (
  id       UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
  name     VARCHAR(128) NOT NULL,
  rtsp     VARCHAR(512) NOT NULL,
  status   VARCHAR(16)  NOT NULL DEFAULT 'offline'
);
CREATE INDEX IF NOT EXISTS idx_cameras_status ON cameras(status);
```

Migration is applied manually with `psql -f`. No migration tool integrated in this scaffold.

## 8. Build, Run, Smoke Test

### `vcpkg.json`

```json
{
  "name": "test-gstreamer",
  "version-string": "0.1.0",
  "dependencies": ["oatpp", "oatpp-postgresql", "oatpp-swagger"]
}
```

### `CMakeLists.txt` (essentials)

```cmake
cmake_minimum_required(VERSION 3.20)
project(test_gstreamer CXX)
set(CMAKE_CXX_STANDARD 20)

find_package(oatpp CONFIG REQUIRED)
find_package(oatpp-postgresql CONFIG REQUIRED)
find_package(oatpp-swagger CONFIG REQUIRED)

add_executable(test_gstreamer src/App.cpp)
target_include_directories(test_gstreamer PRIVATE src)
target_link_libraries(test_gstreamer PRIVATE
    oatpp::oatpp oatpp::oatpp-postgresql oatpp::oatpp-swagger)
target_compile_definitions(test_gstreamer PRIVATE
    OATPP_SWAGGER_RES_PATH="${oatpp-swagger_DIR}/../bin/oatpp-swagger/res")
```

### Configure & build

```bash
cmake -B build -S . -DCMAKE_TOOLCHAIN_FILE=$VCPKG_ROOT/scripts/buildsystems/vcpkg.cmake
cmake --build build
```

### Run

```bash
psql -U postgres -c "CREATE DATABASE test_gstreamer;"
psql -U postgres -d test_gstreamer -f sql/001_init_cameras.sql

./build/test_gstreamer
# Server: http://0.0.0.0:8000
# Swagger UI: http://localhost:8000/swagger/ui
```

### Smoke test

```bash
curl -X POST http://localhost:8000/cameras \
  -H 'Content-Type: application/json' \
  -d '{"name":"Cam 1","rtsp":"rtsp://192.168.1.10/stream","status":"online"}'

curl http://localhost:8000/cameras
```
